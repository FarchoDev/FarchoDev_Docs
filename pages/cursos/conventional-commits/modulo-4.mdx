# Módulo 4: Ejercicios Prácticos

Este último módulo está dedicado a la práctica. Aprenderemos a aplicar los conocimientos adquiridos en los módulos anteriores a través de ejercicios prácticos. La mejor manera de consolidar lo aprendido es ponerlo en práctica.

## Ejercicios Propuestos

1. **Proyecto de ejemplo:**
    * **Creación de un repositorio:** Comenzaremos creando un nuevo repositorio de Git para un pequeño proyecto (por ejemplo, una aplicación web sencilla).
    * **Desarrollo de funcionalidades:** A medida que desarrollemos nuevas funcionalidades, realizaremos commits siguiendo la convención de Conventional Commits.
    * **Corrección de errores:** Simularemos la corrección de errores y realizaremos los commits correspondientes.
    * **Refactorización del código:** Realizaremos cambios en la estructura del código y los documentaremos con commits adecuados.

2. **Revisión de código:**
    * **Análisis de commits existentes:** Analizaremos el historial de commits de un proyecto real (por ejemplo, un proyecto open source) y evaluaremos si se adhiere a las convenciones de Conventional Commits.
    * **Sugerencias de mejora:** Propondremos mejoras en los mensajes de commit y en la estructura del historial.

3. **Generación de un changelog:**
    * **Utilización de herramientas:** Utilizaremos herramientas como `conventional-changelog` para generar un changelog detallado a partir de nuestro historial de commits.
    * **Personalización del changelog:** Adaptaremos el changelog a las necesidades de nuestro proyecto.

## Recursos Adicionales

* **Repositorios de ejemplo:** Te proporcionaré enlaces a repositorios de GitHub que utilizan Conventional Commits de manera efectiva.
* **Plataformas de aprendizaje interactivas:** Exploraremos plataformas como Exercism o Codewars que ofrecen ejercicios prácticos relacionados con Git y Conventional Commits.

## Objetivos del Módulo

* **Consolidar conocimientos:** Aplicar los conceptos teóricos aprendidos en los módulos anteriores a situaciones reales.
* **Desarrollar habilidades:** Adquirir fluidez en la escritura de mensajes de commit convencionales.
* **Experimentar con herramientas:** Familiarizarse con las herramientas disponibles para automatizar tareas relacionadas con Conventional Commits.
