# Módulo 12: Casos de Uso Específicos

En este módulo, veremos cómo aplicar Git a situaciones específicas que pueden surgir en proyectos de desarrollo de software. Exploraremos varios casos de uso y cómo Git puede facilitar la gestión y colaboración en estas situaciones.

## 12.1. Manejo de Proyectos con Múltiples Colaboradores

### Establecer Convenciones de Nombres para Branches y Commits

Cuando trabajas en proyectos con múltiples colaboradores, es importante tener convenciones claras para nombres de branches y mensajes de commit.

- **Nombres de Branches**:

  ```plaintext
  feature/descripcion-de-caracteristica
  bugfix/descripcion-de-correccion
  hotfix/descripcion-de-hotfix
  ```

- **Mensajes de Commit**:

  ```plaintext
  [Tipo] Descripción breve del cambio

  Descripción detallada del cambio, si es necesario.
  ```

### Uso de Pull Requests para Revisiones de Código

Implementa un flujo de trabajo basado en pull requests para garantizar que el código es revisado antes de ser fusionado.

- **Abrir una Pull Request**:

  ```plaintext
  git checkout -b feature/nueva-caracteristica
  # Realiza cambios y realiza commits
  git push origin feature/nueva-caracteristica
  # Abre una pull request en la plataforma de alojamiento del repositorio
  ```

- **Revisar y Aprobar Pull Requests**: Utiliza herramientas de revisión de código en plataformas como GitHub, GitLab o Bitbucket para revisar y aprobar cambios antes de fusionarlos.

## 12.2. Mantenimiento de Proyectos Legacy

### Navegación y Comprensión del Historial

Para proyectos legacy, es importante poder navegar y entender el historial de commits.

- **Ver el Historial de Commits**:

  ```bash
  git log
  ```

- **Ver la Historia de un Archivo Específico**:

  ```bash
  git log -- path/al/archivo
  ```

### Refactorización de Código con Git

Usa branches y commits pequeños y frecuentes para realizar refactorizaciones en proyectos legacy.

- **Crear una Branch para la Refactorización**:

  ```bash
  git checkout -b refactorizacion/nombre-refactorizacion
  ```

- **Realizar Commits Pequeños y Frecuentes**:

  ```bash
  git commit -m "Refactoriza la función X para mejorar la legibilidad"
  ```

- **Fusionar la Branch Refactorizada**:

  ```bash
  git checkout master
  git merge refactorizacion/nombre-refactorizacion
  ```

## 12.3. Gestión de Múltiples Versiones de un Producto

### Uso de Branches para Versiones

Mantén branches separadas para diferentes versiones de tu producto.

- **Crear una Branch para una Nueva Versión**:

  ```bash
  git checkout -b release/v2.0.0
  ```

- **Aplicar Parches a Versiones Antiguas**:

  ```bash
  git checkout release/v1.0.0
  git cherry-pick <commit-id>
  ```

### Etiquetado de Versiones

Usa etiquetas para marcar versiones específicas de tu producto.

- **Crear una Etiqueta para una Versión**:

  ```bash
  git tag -a v2.0.0 -m "Versión 2.0.0"
  ```

- **Enviar la Etiqueta al Repositorio Remoto**:

  ```bash
  git push origin v2.0.0
  ```

## 12.4. Integración de Git con Herramientas de CI/CD

### Configuración de Workflows de CI/CD

Integra Git con herramientas de CI/CD para automatizar pruebas y despliegues.

- **Ejemplo de Workflow de GitHub Actions**:

  ```yaml
  name: CI/CD Pipeline

  on: [push, pull_request]

  jobs:
    build:
      runs-on: ubuntu-latest

      steps:
      - uses: actions/checkout@v2
      - name: Set up Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '14'
      - run: npm install
      - run: npm test
      - run: npm run build
  ```

### Automatización del Despliegue

Configura tu pipeline de CI/CD para automatizar el despliegue de tu aplicación.

- **Ejemplo de Despliegue Automático con GitHub Actions**:

  ```yaml
  name: Deploy

  on:
    push:
      branches:
        - master

  jobs:
    deploy:
      runs-on: ubuntu-latest

      steps:
      - uses: actions/checkout@v2
      - name: Set up Node.js
        uses: actions/setup-node@v2
        with:
          node-version: '14'
      - run: npm install
      - run: npm run build
      - name: Deploy to Server
        run: |
          scp -r build/* user@server:/path/to/deploy
  ```

## Resumen

En este módulo, hemos explorado casos de uso específicos para Git, incluyendo:

- **Manejo de Proyectos con Múltiples Colaboradores**: Convenciones de nombres, uso de pull requests.
- **Mantenimiento de Proyectos Legacy**: Navegación del historial, refactorización de código.
- **Gestión de Múltiples Versiones de un Producto**: Uso de branches y etiquetas.
- **Integración de Git con Herramientas de CI/CD**: Configuración de workflows y automatización del despliegue.

Estos casos de uso te ayudarán a aplicar Git de manera efectiva en diferentes situaciones, mejorando la colaboración, el mantenimiento y la gestión de versiones en tus proyectos.
