# Módulo 9: Gestión de Versiones Semánticas y Etiquetado en Git

En este módulo, exploraremos cómo gestionar versiones de tu proyecto utilizando el versionado semántico (SemVer) y cómo etiquetar (tagging) en Git para marcar versiones específicas de tu código.

## 9.1. Versionado Semántico (SemVer)

El versionado semántico es una convención para asignar números de versión a tu software. Los números de versión siguen el formato `MAJOR.MINOR.PATCH`, donde cada parte tiene un significado específico:

1. **MAJOR**: Se incrementa cuando realizas cambios incompatibles en la API.
2. **MINOR**: Se incrementa cuando añades funcionalidad de manera compatible con versiones anteriores.
3. **PATCH**: Se incrementa cuando realizas correcciones de errores compatibles con versiones anteriores.

### Ejemplo de Versionado Semántico

- **1.0.0**: Primera versión estable.
- **1.1.0**: Añade nueva funcionalidad compatible.
- **1.1.1**: Corrección de errores en la funcionalidad existente.

## 9.2. Creación de Etiquetas (Tags) en Git

Las etiquetas en Git se utilizan para marcar puntos específicos en el historial del repositorio. Las etiquetas son útiles para marcar versiones de lanzamiento, puntos importantes en el desarrollo, o cualquier otro hito significativo.

### Tipos de Etiquetas

- **Etiquetas Anotadas**: Incluyen información adicional como el nombre del creador, la fecha y un mensaje. Son almacenadas como objetos completos en la base de datos de Git.
- **Etiquetas Ligeras**: Son simplemente referencias a commits específicos y no incluyen información adicional.

### Crear una Etiqueta Ligera

```bash
git tag <nombre-de-la-etiqueta>
```

Ejemplo:

```bash
git tag v1.0.0
```

### Crear una Etiqueta Anotada

```bash
git tag -a <nombre-de-la-etiqueta> -m "Mensaje de la etiqueta"
```

Ejemplo:

```bash
git tag -a v1.0.0 -m "Primera versión estable"
```

### Listar Etiquetas

Para ver todas las etiquetas en tu repositorio, usa:

```bash
git tag
```

### Mostrar Información de una Etiqueta Anotada

Para ver información detallada sobre una etiqueta anotada:

```bash
git show <nombre-de-la-etiqueta>
```

### Compartir Etiquetas con Remotos

Por defecto, las etiquetas no se envían automáticamente al repositorio remoto cuando haces un `git push`. Necesitas enviarlas explícitamente:

1. **Enviar una Etiqueta Específica**

   ```bash
   git push origin <nombre-de-la-etiqueta>
   ```

   Ejemplo:

   ```bash
   git push origin v1.0.0
   ```

2. **Enviar Todas las Etiquetas**

   ```bash
   git push origin --tags
   ```

### Eliminar una Etiqueta

Para eliminar una etiqueta localmente:

```bash
git tag -d <nombre-de-la-etiqueta>
```

Ejemplo:

```bash
git tag -d v1.0.0
```

Para eliminar una etiqueta del remoto:

```bash
git push origin --delete <nombre-de-la-etiqueta>
```

Ejemplo:

```bash
git push origin --delete v1.0.0
```

## 9.3. Ejemplos Prácticos de Etiquetado

### Marcando una Versión de Lanzamiento

1. **Crear una Etiqueta Anotada**

   ```bash
   git tag -a v2.0.0 -m "Segunda versión mayor con nuevas funcionalidades"
   ```

2. **Enviar la Etiqueta al Remoto**

   ```bash
   git push origin v2.0.0
   ```

### Listar y Mostrar Etiquetas

1. **Listar Todas las Etiquetas**

   ```bash
   git tag
   ```

2. **Mostrar Información de una Etiqueta Anotada**

   ```bash
   git show v2.0.0
   ```

## Resumen

En este módulo, has aprendido cómo gestionar versiones y etiquetar puntos específicos en tu repositorio Git:

- **Versionado Semántico:** Utilizar `MAJOR.MINOR.PATCH` para gestionar versiones.
- **Etiquetas en Git:** Crear, listar, mostrar y eliminar etiquetas, así como compartirlas con remotos.
- **Etiquetas Anotadas y Ligeras:** Entender las diferencias y cuándo utilizar cada tipo.

El versionado semántico y el etiquetado son herramientas esenciales para la gestión de versiones de tu proyecto, permitiéndote mantener un historial claro y organizado de los lanzamientos y cambios significativos en tu código.
